# Story 9.2: Region Management

As a **Platform Administrator**,
I want to view, create, edit, and manage regions,
So that I can configure the geographic foundation for all plantation data.

## Acceptance Criteria

**AC 9.2.1: Region List View**

**Given** I navigate to `/regions`
**When** the page loads
**Then** I see a card grid of all regions with:
  - Region name and county
  - Altitude band badge (Highland/Midland/Lowland)
  - Factory count
  - Farmer count (aggregated)
  - Status indicator (Active/Inactive)
**And** I can filter by county and altitude band
**And** I can search by region name

**AC 9.2.2: Region Detail View (Top-Level)**

**Given** I click on a region card
**When** the region detail page loads
**Then** I see the region information panel (editable)
**And** I see a read-only summary of factories in this region (count, names)
**And** I see a read-only summary of farmers in this region (count)
**And** factory links navigate to the Factories screen (filtered by region)

**AC 9.2.3: Region Creation**

**Given** I click "+ Add Region"
**When** I complete the form
**Then** I provide:
  - Name, County, Country (default: Kenya)
  - **Region boundary via interactive map** (polygon drawing, circle tool, or GeoJSON import)
  - Altitude band (Highland/Midland/Lowland)
  - Flush calendar (first flush, monsoon, autumn, dormant periods)
  - Agronomic factors (soil type, diseases, harvest hours, frost risk)
  - Weather API configuration
**And** the map displays existing factories/CPs as reference points
**And** the system auto-calculates center point (centroid) and area from the drawn boundary
**And** the region is created with status Active
**And** I'm redirected to the new region detail page

**AC 9.2.3a: Map-Based Boundary Definition**

**Given** I am on the region creation or edit form
**When** I interact with the boundary map component
**Then** I can:
  - **Draw Polygon**: Click points on the map to define an irregular boundary (double-click to complete)
  - **Draw Circle**: Click center + drag to define radius (for simple circular regions)
  - **Import GeoJSON**: Upload official boundary files from government/GIS sources
  - **Clear**: Remove current boundary and start fresh
**And** the map shows:
  - Satellite/terrain toggle for visual reference
  - Existing factories (ğŸ­) and collection points within view
  - Real-time area calculation (kmÂ²) and perimeter display
  - Zoom controls and search-by-location
**And** I can edit the boundary by dragging vertices after drawing

**AC 9.2.4: Region Editing**

**Given** I'm on a region detail page
**When** I click "Edit"
**Then** the region fields become editable inline
**And** I can modify any field
**And** changes are saved when I click "Save"
**And** I can cancel to discard changes

**AC 9.2.5: Weather Observations Display**

**Given** I'm on a region detail page
**When** the page loads
**Then** I see the last 7 days of weather observations for this region:

- Date, Temperature (min/max), Precipitation (mm), Humidity (%)

**And** weather alerts are highlighted with indicators:

## Technical Notes

- **BFF endpoints** (provided by Story 9.1c): `GET/POST /api/admin/regions`, `GET/PUT /api/admin/regions/:id`
- **This story is UI-only** - consumes APIs created in Story 9.1c
- Backend: Plantation Service gRPC `ListRegions`, `CreateRegion`, `UpdateRegion`, `GetRegion`
- Region ID auto-generated: `{county}-{altitude_band}` (e.g., `nyeri-highland`)
- **Boundary Storage**: GeoJSON Polygon format in MongoDB
  ```json
  {
    "boundary": {
      "type": "Polygon",
      "coordinates": [[[-0.42, 36.95], [-0.41, 36.96], ...]]
    },
    "centroid": { "lat": -0.4197, "lng": 36.9553 },
    "area_km2": 342.5,
    "perimeter_km": 78.2
  }
  ```
- **Map Implementation**: See [ADR-017: Map Services and GPS-Based Region Assignment](architecture/adr/ADR-017-map-services-gps-region-assignment.md)
  - **Library**: Leaflet.js + react-leaflet (zero cost, offline-capable)
  - **Tiles**: OpenStreetMap (free, no API key required)
  - **Drawing**: Leaflet.draw plugin for polygon/circle tools
  - **Calculations**: Turf.js for area, perimeter, centroid computation
  - **Component**: Use `<BoundaryDrawer>` from ADR-017 Section 4.4
